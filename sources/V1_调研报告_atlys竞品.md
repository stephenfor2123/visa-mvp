## §1 封面（决策点：这份 deck 解决什么问题）

**为什么**：用户在 Atlys 之外做一款"中国 + 发展中国家 → 欧美日澳签证"的 App，要一份能立刻拿去找合伙人 / 投钱 / 排 6 个月节奏的内部手册。这份文件把 Atlys 的产品骨架拆成业务 / 技术 / 合规 / 经济模型 / 落地路径五张可填空的表，再倒推出 MVP 节奏与首批种子用户的投放节奏。

**核心结论（3 个最关键决策）**：

1. **MVP 不先打中国大陆**。iOS 签证代办类目上架需出入境中介资质 + 类目审核，先打 **印度 + 印尼 + 越南**（Tier 1），6 个月后扩巴西 / 墨西哥 / 中亚（Tier 2），12 个月后回流中国大陆。
2. **按效果付费不是自保险**。是 **第三方拒签险 + 服务费 + 保险佣金** 的三方套利：拒签险打包进客单价，向保险公司采购，再向用户承诺"拒签退部分服务费"。拒签率 8% 是行业上界，App 自身不赔。
3. **6 个月、4 个里程碑、8–12 人、100–150 万人民币**。第 12 个月中性情景 ARR $1.3M、单订单净利 $3.5、12 个月回收 CAC。

**风险提示**：
- 拒签率 / 保险赔付率 / App Store 拒审 / GDPR 跨境 / 印度 OTA MakeMyTrip 是 Top-5 风险，必须有 Owner + 缓释动作。
- 创始人节奏（M6 团队 12 人）是隐性风险——M2 后必须引入 COO / 运营合伙人。
- "一键生成单据"必须走"真实第三方预订"白名单路径，**任何伪造 / PS / OCR 改写 = 红线**。

**行动项**（谁 / 什么时候 / 怎么验收）：
- **CEO**：1 周内决定是否按"印度 Schengen 单点 + 拒签险单产品 + 1 个 BD"极简组合起盘。
- **CTO**：1 周内完成 Flutter 跨平台 / 云区域（AWS Singapore + Mumbai）选型，2 周出技术 PoC。
- **CFO/法务**：2 周内完成平安拒签险接入询价 + DPO（OneTrust/TrustArc）比价。

### 4.3 中国大陆策略（重点）

- **App Store / 国内应用市场上架**：
  - 主体公司需 **出入境中介机构资格认定**（公安部出入境管理局）。
  - iOS 类目选"旅游 / 商务 / 工具"，**禁选**"金融 / 移民"。
  - 内容审核：禁"代传虚假材料" / "包过" / "100% 出签"。
- **建议 MVP 不先上大陆**：
  - 原因：上架周期 2–4 周 / 合规成本高 / 7+ 平台分发限制。
  - 替代：先做 **香港（iOS HK 区） + 东南亚（iOS SG/PH/ID 区） + 拉美（iOS MX/BR 区）** 上架。
  - 路径：自然流量测试 6 个月，再回流大陆上架。

## §5 业务模型与拒签险套利结构（核心差异化）

**为什么**：用户原话"按效果付费 + 我把 app 的收入去买了保险"——这是整个模型的发动机。必须先讲清**不是自保险**，否则后面的 ROI 全错。

### 5.1 三层结构

1. **拒签保险产品**（第三方）：平安"签证拒签补偿保险"亚洲 200 元 / 全球 500 元定额；保费 40/70 元；保险期 ≤60 天。
2. **App 服务费**：用户支付"签证服务费"，其中一部分用于"向保险公司批量采购拒签险"，打包率 70%–90% 视为可接受。
3. **拒签后退费承诺**：App 公开承诺"未签即退服务费 30%–50%"，理赔风险通过拒签险向保险公司转移。

### 5.2 现金流图（中性情景）

```
用户支付 $50 服务费
    ├─→ App 净得 $44（扣 OTA Affiliate 渠道分润）
    ├─→ 拒签险采购 $6（向平安/Allianz 采购）
    └─→ 拒签后：
         ├─ 保险公司赔 $200 / $500 → 用户拿
         └─ App 退 30% 服务费 = $15 → 用户拿
         ⇒ App 拒签后净亏 = $15（服务费退） + $0（保险已转嫁）
         ⇒ App 真实拒签成本仅 = 退费 + 客服时间 ≈ $18
```

### 5.3 三档情景下的单订单净利

| 情景 | 拒签率 | 客单价 | 拒签险摊销 | 退费支出 | 单订单净利 |
|---|---|---|---|---|---|
| 乐观 | 5% | $55 | $4 | $0.8 | **$9.2** |
| 中性 | 8% | $50 | $6 | $1.5 | **$3.5** |
| 悲观 | 15% | $45 | $9 | $3.0 | **-$3.6** |

### 5.4 关键敏感度
- 拒签率每 +1pp → 单订单净利 -$0.6
- 客单价每 +$5 → 单订单净利 +$4
- CAC 每 -$1（投放优化） → 单订单净利 +$1

### 5.5 红线 / 雷区
- **不承诺"100% 拒签全退"**——构成赌博式承诺 / 虚假宣传。
- **不打"包过"广告**——触发 App Store 4.3 条款 + iOS 类目审核拒绝。
- **不与"代传虚假材料"挂钩**——触发 10 年全球拒签 + 刑事责任。

### 5.6 必备用户协议条款
- "本平台提供的预订确认仅用于签证申请辅助，不保证申请结果。"
- "任何资料造假由申请者本人承担法律责任。"
- "未成年人代办须由监护人现场视频确认 + 公证文件。"

- **iOS**：
  - 隐私政策强制（个人数据收集前）

- **单点故障**：团队只有 1 个 Flutter 资深，M3 前必须再招 1 个 Flutter 或 1 个 iOS。

## §8 GDPR & 数据合规（5 大机制 + DPO）

**为什么**：用户主动问 GDPR，说明他既想打申根又怕翻车。这页是给法务和投决看的"安心丸"。

### 8.1 适用判定
- 任何处理 EEA 居民个人数据的全球企业都受 GDPR 约束，与公司是否在 EEA 无关。
- 申根签证申请场景中，申请者若为欧盟永居 / 欧盟居民，即触发 GDPR。
- 中国大陆《个人信息保护法》（PIPL）并行适用，跨境数据出境需安全评估。

### 8.2 5 大落地机制

| 机制 | 内容 | Owner | 工具 |
|---|---|---|---|
| **DPA** | 与所有数据处理方（云、OCR、支付、保险、Affiliate）签 DPA | 法务 | 模板（IAPP） |
| **SCC** | 跨境数据传输必须用 EU 2021 标准合同条款 | 法务 | 模板（EU 委员会） |
| **TIA** | 转移影响评估（Transfer Impact Assessment） | DPO | 模板（IAPP） |
| **PIA** | 每次新增敏感数据（护照 / 签证 / 人脸 / 指纹 / 行程）前做 PIA | DPO + 产品 | 模板（CNIL） |
| **同意管理** | iOS 14+ ATT / Android 13+ 隐私沙箱 / Cookie banner | 前端 | OneTrust |

### 8.3 用户权利实现（App 内 UI）
- **访问权**：个人数据导出 JSON / CSV
- **更正权**：护照、地址、邮箱可改
- **删除权**（被遗忘）：账号注销 30 天后彻底删除（含 S3）
- **限制处理权**：用户可暂停"营销推荐"
- **数据可携**：导出 PDF + JSON
- **反对自动化决策**：人工客服接管

### 8.4 DPO 设立门槛
- 大规模处理敏感数据 → 强制设 DPO。
- 签证 App 处理护照 / 指纹 / 人脸属于"大规模敏感数据处理"——**默认要设 DPO**。
- **推荐外包**：OneTrust / TrustArc / IAPP 注册 DPO 服务（年费 $5K–$20K）。

### 8.5 罚则与先例
- 最高 4% 全球年营收或 €20M（取高者）。
- 实际案例：Meta 2023 被罚 €12 亿；TikTok 2023 被罚 €345M（儿童数据违规）。

## §12 风险登记册 + 多 Agent 复盘

**为什么**：这是给所有合伙人 / 投决看的"清醒剂"。每条风险有 Owner + 缓释动作。

### 12.1 Top 8 风险登记册

| # | 风险 | 等级 | Owner | 缓释动作 |
|---|---|---|---|---|
| 1 | 拒签率波动 >15% | 高 | COO | 拒签险 + 再保 + 服务费部分退 |
| 2 | App Store 拒审（iOS 签证代办类） | 高 | 法务 + 产品 | 不上"代办"用词；类目选"工具 / 旅游" |
| 3 | GDPR / PIPL 双轨违规 | 高 | DPO（外包） | DPA + SCC + PIA 制度化 |
| 4 | OTA Affiliate API 被封 | 中 | BD | 同时接 Booking + 万豪 + 自建 PNR 备份 |
| 5 | 创始人节奏（M6 团队 12 人） | 中 | CEO | M2 后引入 COO + 运营合伙人 |
| 6 | 拒签险保险公司拒赔 | 中 | CFO | 拒赔条款 100% 透明公开，避免承诺过度 |
| 7 | 竞品 Atlys 跟进印度 / 东南亚 | 中 | CEO | 差异化：拒签险套利 + 多语 + 多目的地 |
| 8 | 出入境政策突发变化（如美签 $250 完整性费） | 中 | 合规 | 多目的地分散 + 政策监测机器人 |

### 12.2 多 Agent 复盘（4 视角 / 20+4 条）

#### 12.2.1 业务可行性视角（Founder / Operator）
**定性**：MVP 12 个月跑通单点经济模型有戏，但 6 个月就跑通乐观路径需要"印度 Schengen 单点 + 拒签险单产品 + 1 个 BD"的极简组合。

**5 条挑刺**：
1. 拒签险的赔付率数据在平安 / 太平洋都未公开，没有 benchmark 怎么定价？
2. 拒签率 8% 的口径是"申请者层面"，但 App 内会聚集"高风险申请者"（首次、白本、单身）——实际拒签率可能 12%–15%。
3. 印度 OTA MakeMyTrip 已经是 Atlys 股东，资源端被锁；只能打"非 MakeMyTrip 渠道"（线下旅行社 + 直客）。
4. iVisa 在 iOS US 旅游榜 316 名、5,276 评分，5 年只做这个市场已经到顶——是不是赛道天花板？
5. 创始人 6 个月排 4 个里程碑太紧；M2（5,000 单）很可能滑到 M3。

**1 条亮点**：
- **"拒签险套利"是真创新**：Atlys 没做，平安拒签险单价低、保额小，App 把"用户→拒签险→保险公司"链路重排，**每单多赚 $3–$5 纯佣**——这是别的 App 看不上的钱。

#### 12.2.2 合规与法律视角（Legal / Privacy Officer）
**定性**：MVP 设计绕开"代办"用词、改走"信息聚合 + 工具"路线，可以过审；但 iOS 16+ 隐私新政、未成年人代办、跨境数据三条线必须前置合规。

**5 条挑刺**：
1. "一键生成行程单"必须明确"来自真实第三方预订"——否则被认定"协助造假"，触发 App Store 4.3 条款（spam）。
2. 中国大陆主体公司需出入境中介资质；若主体在境外，App 内不能出现"中国大陆签证服务"字样，但可以为"已在中国大陆用户"提供"赴东南亚签证"服务。
3. EEA 居民数据即便不存欧盟，也必须 SCC + TIA，不能用"我们在新加坡所以不受 GDPR"的解释。
4. 拒签险产品本身要保监备案（如使用境内保险），跨境保险（如 Allianz）需要再保牌照或合作代理。
5. 18 岁以下未成年人代办——必须监护人现场视频确认 + 公证 + 双重 KYC，不能纯线上。

**1 条亮点**：
- **"拒签退部分服务费"的承诺比"全退"合规得多**——"保险 + 部分退"组合既不构成赌博式承诺，也不构成虚假宣传，是合规边界内最优的折中。

#### 12.2.3 技术落地视角（CTO / Lead Engineer）
**定性**：6 个月 4 里程碑、Flutter 跨平台、8–12 人是可行配置；但 OCR 准确率 / Affiliate OTA 接入 / 拒签险 API 三处是隐藏技术债。

**5 条挑刺**：
1. OCR 护照识别准确率官方 95%——但发展中国家护照（孟加拉、尼日利亚）实际可能 88%，需要"人工二次审核"工作流预留。
2. Booking / 万豪 Affiliate 不一定开放"未付款可取消"批量 API，需要 BD 单独谈合作。
3. 拒签险 API 国内（平安）和海外（Allianz）两套数据模型不一致，状态机设计复杂。
4. iOS 17+ Live Activity / Android 14+ 通知权限收紧，签证状态推送需要重新设计。
5. 团队只有 1 个 Flutter 资深——单点故障；M3 之前必须再招 1 个 Flutter 或 1 个 iOS。

**1 条亮点**：
- **Flutter 跨平台 + 中国境外云（AWS Singapore/Mumbai）的组合**，让"中国团队服务全球用户"在 6 个月内成为可能；如果是 RN，包体小但稳定性差。

#### 12.2.4 增长与 ROI 视角（Growth Lead / CFO）
**定性**：CAC 中性 $12 / 12 个月回收是合理承诺；但 1 万种子用户阶段，投放成本会被 KOL 报价扭曲，需要早期就建数据反馈环。

**5 条挑刺**：
1. 印度 KOL 单条报价 $200–$2,000，1 万种子用户如果全靠 KOL，CAC 飙到 $20+。
2. 拒签率 8% 看起来可控，但 1 单拒签 = App 退 30% 服务费 + 拒签险赔保额 + 客服时间，单次拒签 App 真实成本 ≈ $15。
3. 拒签险佣金 20% 是行业上界，平安实际能给到 12%–15%，要再谈。
4. 12 个月回收期对种子轮已经太长，对天使轮勉强——可能要 18 个月回收才现实。
5. Affiliate OTA 收入（Booking / 酒店分佣）能否覆盖 CAC 缺口？需要单独算 OTA 增量 ARPU。

**1 条亮点**：
- **"拒签险打包率 70%+" 是隐形 ARR**：每多 1% 打包率，年化净利 +$2 万——靠产品和文案就能拉起来，几乎零成本。

### 12.3 复盘结论：4 视角共识
1. **6 个月单点跑通可以，10 个月规模跑通比较现实**。
2. **拒签险是真差异化，但合规 + 客服 + 数据治理要前置 2 个月**。
3. **不能打"代办 / 包过"——必须走"信息聚合 + 工具"路线**。
4. **中国大陆不是 MVP 的起点，是 12 个月后的回流点**。
