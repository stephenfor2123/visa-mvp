# 旅游签引流快速评分 V5

## 定位

- 目标：用户 1 分钟内完成测试并进入办理流程。
- 范围：仅旅游签；美国、英国、申根、澳大利亚。
- 形式：6 个问题，固定加减分，0–100 分。
- 口径：页面可使用“通过率高/中等/较低”，底部注明“快速评估，仅供参考，不代表使领馆决定”。
- 语言：只改变文案和收入档位显示，不改变同一标准档位的分值。

## 题目

1. 婚姻状况：未婚 / 已婚 / 离异 / 丧偶。
2. 月收入：5 档；不同语言显示当地币种，但传统一枚举。
3. 近 5 年国际旅行次数：无 / 1–3 / 4–10 / 10+。
4. 签证历史：从未办理 / 1–2 次正常签证 / 3+ 次正常签证 / 有拒签记录。
5. 当前工作状态：在职 / 自由职业 / 学生 / 退休 / 待业。
6. 附加信息：年龄；是否单身女性独自旅行。

旅游目的固定为 `tourism`，不显示选择按钮。

## 计算

```text
score = clamp(58
  + marital_points[country][answer]
  + income_points[country][answer]
  + travel_points[country][answer]
  + visa_points[country][answer]
  + employment_points[country][answer]
  + age_points[country][age, marital]
  + solo_female_points[country][checked], 0, 100)
```

等级：

- 75–100：通过率高；
- 50–74：通过率中等；
- 0–49：通过率较低。

- 30–50 岁：+3；

- 单身女性独自旅行：美国 −8、英国 −6、申根 −5、澳洲 −6。

## 多语言收入档

前端可显示当地更直观的金额，但必须传以下统一枚举：

```text
low / lower / middle / upper / high
```

切换语言时不重新评分、不自动改变已经选择的档位。语言与用户常住国不同的情况下，优先使用用户选择的收入币种；MVP 没有币种选择时使用 locale 对应显示档位。
