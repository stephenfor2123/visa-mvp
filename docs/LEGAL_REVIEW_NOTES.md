## 0. 谁签署 18 项 checklist？（直接回答）

**不是产品负责人自签就能过关。**

| 角色 | 能做什么 | 不能做什么 |
|------|----------|------------|
| **你（产品 / 创始人）** | 确认业务事实（服务器在哪、用哪些第三方、是否允许未成年人注册）；把 checklist 交给律师；在律师出具意见后**确认已收到意见并决定是否上线** | 用自己的勾选代替「法务通过」 |
| **法务 / 外部律师** | 对 checklist 逐项给出 **通过 / 需修改 / 不通过**，并**签字或邮件确认** | — |
| **印尼 / 越南本地律师** | 审 id / vi 全文 + UU PDP / Nghị định 13 专项（checklist #2/#3/#6/#14/#17/#18） | — |
| **工程 / AI** | 起草文案、对齐 GDPR 框架、更新状态标记 | **不能**把状态改成「法务已核」 |

文档 §3 写明：

> 每一项需**法务**在 4 语种下都明确给出「通过 / 需修改 / 不通过」结论，并**签字 / 邮件确认**。

你作为负责人的动作是：**发包给律师 → 收回意见 → 让工程改文案 → 再请律师复核 → 勾选「法务已核」**。你本人勾选「我看过了」只算业务确认，**不算**法务签核。

## 2. GDPR 标准对齐（本次已写入文案的部分）

### 2.1 跨境数据传输 — GDPR Chapter V (Arts. 44–49)

| GDPR 要求 | 本版文案如何写 | 仍需法务/业务拍板 |
|-----------|----------------|-------------------|
| 禁止无保障的跨境传输 (Art.44) | 声明跨境须有适当保障 | 生产托管地区最终选址 |
| 适当保障 (Art.46)：SCC / DPA | 写明 SCC 或同等 DPA | 与云厂商 / Stripe / OCR 签真实 DPA |
| 充分性决定 (Art.45) | 未假装已有充分性 | 若选欧盟用户，再评 adequacy |
| 数据最小化 + 目的限制 | 已写入 §3 | — |
| 使领馆提交 | 明示签证申请会向目的地国机关跨境提供 | — |
| 撤回非强制跨境同意 | privacy@htex.app | 运营流程落地 |
| 印尼 UU PDP / 越南 NĐ 13 | 文案引用当地机制 | **本地律师确认报备/DPIA 是否必须** |

**不能只靠文案完成的**：真实服务器司法管辖区、SCC 签约、印尼 PDP 报备、越南 DPIA/备案。

### 2.2 未成年人 — GDPR Art. 8

| GDPR 要求 | 本版做法 |
|-----------|----------|
| 信息社会服务同意默认 **16 岁**（成员国可降至 ≥13） | 平台统一 **16 岁**门槛 |
| 未达年龄需监护人同意 | 用户协议 §6 + 隐私 §6：未满 16 不得自注册；未成年人办签须监护人代操作 |
| 合理努力核实 | 文案层已声明；**技术侧年龄门控仍待产品实现** |
| 印尼 / 越南儿童数据 | 文案写「当地监护人同意要求从其规定」 |

### 2.3 同意强度 — Recital 32 / Art. 7

- 勾选：`已阅读、理解并同意`（四语种）
- 注册框须**默认未勾选**（工程侧已按此设计，见 checklist #10）
- 敏感材料上传另有明示同意弹窗（见 `docs/PRIVACY_COMPLIANCE.md`）

### 2.4 用户权利 — Arts. 12–22

- 访问 / 更正 / 删除 / 限制 / 反对 / 撤回同意 / 可携（适用时）
- SLA：**15 个工作日**回复（写入隐私 §5）
- 注销：72 小时宽限期后永久删除（产品已实现）

## 4. Review 完成后的状态变更流程

```
v1.1-gdpr-draft ──律师 review──> 需修改则改文案 ──律师签字/邮件──> 法务已核 ──launch──> 生产生效
                                                                    (移除 WARNING / pending 标记)
```

- **法务已核**: 删除 4 语种 `__legal_review_pending__`；删除 `Agreement.vue` 中 `LEGAL_TODO`；移除生产 WARNING banner
- **拒绝**: 律师出 notes → 工程改 → 再进 review

**禁止**: 未经签字把状态改成「法务已核」或删掉 pending 标记。
