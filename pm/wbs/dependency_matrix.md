## 3. 阻塞升级路径

```
A 端 写代码卡住
  → 先看本矩阵表:卡的是谁?B 的对应任务做完没?
    → 已做完: 拉 B 一起 debug (15min 内解不掉就升)
      → 15min 解不掉 → 升 PM (D)
        → PM 拉 CTO/相应 Lead (30min 内出方案)
          → 30min 还没解 → 写进 risk register,改 WBS,可能延期
```

**原则**:
- 15 分钟没思路,先问人;30 分钟没解,升级 PM
- 升级后 PM 必须在 4 小时内给方案或风险登记
- 不允许"沉默卡 1 天",沉默即事故

## 4. 关键路径 (Critical Path)

> 任何 Critical Path 任务延期,直接导致上线日 M8 (2026-08-07) 延期.

```
infra1 (服务器 06-15) → infra2 (Docker 06-18) → be1 (后端脚手架 06-19)
  → be2 (Auth+Users 06-29) → be3 (Orders+RPA 07-09) → be4 (Admin 07-17)
    → fe5 (后台 07-15) → qa3 (集成测试 07-03~07-10) → qa4 (性能压测 07-20~07-24)
      → qa5 (UAT 07-27~07-31) → qa6 (上线 Gate 08-03~08-07)
```

**8 周关键路径长度**:约 38 工作日,buffer 约 2 天(7/25 周五前后). 任何主要任务延 2 天,buffer 吃光.
